AJAX Search and Sort ReadMe
XMod Pro User Group, March 2015

by Kelly Ford DNNDev.com

Building an AJAX-based searching and sorting solution leveraging XMod Pro. XMod Pro is well-known for its ability to create data-based solutions quickly. It has built-in searching and sorting that doesn't require you to know any programming or any SQL. In this solution, however, I develop an AJAX searching and sorting method. For this, you'll need to create a template to display your data, a feed to communicate with the database, and a stored procedure to perform the search/sort and return the appropriate data.

This solution also relies on jQuery for communication with the feed and KnockoutJS for data-binding. The project uses bootstrap for presentation and layout. In addition, the solution references an older version of Wijmo, an open-source UI framework. The Wijmo pager widget is used to provide paging for the solution. Bootstrap and Wijmo are used primarily to quickly build the UI and are not required for this solution. They can be swapped out with whatever solutions work best for you.

Since this is a fairly advanced subject, I assume you know your way around XMod Pro and SQL Server Management Studio - enough to be able to execute SQL scripts.

Installation & Setup

  1. Create the Table and Sample Data: I have included data.sql in the /db folder.
    1. Edit the SQL Script: Edit the first line of the file where it says: 
      USE [replace_with_your_database_name]
    2. Change replace_with_your_database_name to be the name of the database you're installing into.
    3. Save and Run the Script in SQL Server Management Studio

    This will create the Contacts table and populate it with 200 rows of data
    This script will create a table named Contacts. If your database already has a table named Contacts, you should receive an error indicating the object already exists, but don't rely on that. Make sure your data is backed up before any installation.
  2. Create the Search/Sort Stored Procedure:
    1. Open the /db/sproc.sql file in your favorite text editor.
    2. Edit the SQL Script: Edit the first line of the file where it says: 
      USE [replace_with_your_database_name]
    3. Change replace_with_your_database_name to be the name of the database you're installing into.
    4. Save and Run the Script in SQL Server Management Studio
    This will create the Contacts table and populate it with 200 rows of data
    This script will create a stored procedure named sp_Select_Contacts. If your database already has a stored procedure named sp_Select_Contacts, you should receive an error indicating the object already exists, but don't rely on that. Make sure your data is backed up before any installation.
  3. Create the Feed: Create a new Feed named getContacts. Remove any auto-generated code and replace it with the content of the file search_sort_feed.txt
  4. Create the Template: Create a new Template named Contacts (or choose a name appropriate for your site). Remove any auto-generated code and replace it with the content from the file search_sort_template.txt
  5. Install the Solution Files: This solution was built in a development DNN website. In the root of the website I created the following folder structure: 
    /usergroup/2015/03
    Copy the /css, /fonts, /img folders and their contents into this folder on your site.

    If you want to place these files in a different location, you will just need to edit your Template file to reference the new location.

Package Contents

  • read_me.html: This file
  • search_sort_template.txt: The XMod Pro Template. To use it, create a new blank template and copy and paste the contents of this file into it.
  • search_sort_feed.txt The XMod Pro Feed used in this solution. To use it, create a new blank feed with the name "getContacts" (withouth quotes) and past the contents of this file into it.
  • /db/data.sql: SQL Script that will create the Contacts table and populate it with random data.
  • /db/sproc.sql: SQL Script that will create the stored procedure from this project.
  • /img folder: images used by they template
  • /fonts folder: Font Awesome fonts used in the solution
  • /css folder: Styling used by the template. The style.css file contains styling for the table
